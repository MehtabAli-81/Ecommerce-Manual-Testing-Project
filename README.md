
# 🛒 E-Commerce Manual Testing Project




## 📌 Project Overview
This project demonstrates a complete Manual Testing lifecycle for an E-Commerce application. It includes all major testing artifacts such as Test Plan, Test Scenarios, Test Cases, Bug Reports, and RTM (Requirement Traceability Matrix).

The goal of this project is to simulate real-world QA practices and showcase structured testing documentation for an E-Commerce system.
## 🎯 Objectives
To understand and apply manual testing concepts in a real-world scenario.

To design effective test scenarios and test cases.

To identify and document defects.

To ensure requirement coverage through RTM.

To build a professional QA portfolio project.
## 🧪 Scope of Testing
The following modules of an E-Commerce website are covered:

User Registration & Login

Product Browsing & Search

Product Details Page

Add to Cart

Cart Management

Checkout Process

Payment Flow (Simulated)

Order Confirmation

## 📄 Deliverables
1. Test Plan

Contains:

Objective of testing
Scope and approach
Resources and environment
Testing strategy
2. Test Scenarios

High-level scenarios covering all major functionalities of the application.

3. Test Cases

Detailed test cases including:

Test Case ID
Description
Preconditions
Steps
Expected Result
Actual Result
Status
4. Bug Reports

Documented defects with:

Bug ID
Description
Severity & Priority
Steps to reproduce
Expected vs Actual result
5. RTM (Requirement Traceability Matrix)

Maps requirements to corresponding test cases to ensure full coverage.

6. Screenshots

Visual evidence of test execution and defects.
